  <name>Task 1: Salesforce API wrapper (sf_api.py)</name>
  <files>migrate/sf_api.py, tests/test_sf_api.py</files>
  <behavior>
    - build_client(creds: Credentials) returns a simple_salesforce.Salesforce instance using access_token and instance_url (session_id=access_token, instance_url=instance_url)
    - query_all(client, soql: str) returns list[dict] by consuming client.query_all(soql)["records"], stripping the "attributes" key from each record
    - insert_records(client, sobject: str, records: list[dict]) calls client.bulk.{sobject}.insert(records) and returns the list of results; each result has {"id": str, "success": bool}
    - get_record_type_map(client, sobject: str) queries RecordType where SobjectType=sobject, returns dict mapping DeveloperName -> Id
  </behavior>

Verify query_all strips "attributes" from records. Verify insert_records delegates to bulk API. Verify get_record_type_map returns {DeveloperName: Id} dict.

    Then create migrate/sf_api.py implementing:
    - build_client(creds: Credentials) -> Salesforce: Construct with session_id=creds.access_token, instance_url=creds.instance_url
    - query_all(client: Salesforce, soql: str) -> list[dict]: Run client.query_all(soql), iterate records, strip "attributes" key from each, return list
    - insert_records(client: Salesforce, sobject: str, records: list[dict]) -> list[dict]: Call client.bulk.{sobject}.insert(records), return results. Use getattr(client.bulk, sobject) for dynamic SObject access.
    - get_record_type_map(client: Salesforce, sobject: str) -> dict[str, str]: Query "SELECT Id, DeveloperName FROM RecordType WHERE SobjectType = '{sobject}'", return {r["DeveloperName"]: r["Id"] for r in results}

<task type="auto" tdd="true">
  <name>Task 2: ETL helpers (etl.py) — skip logic, RecordType remap, two-pass insert</name>
  <files>migrate/etl.py, tests/test_etl.py</files>
  <behavior>
    - extract_records(client, sobject_api_name: str, fields: list[str]) builds SOQL "SELECT {fields} FROM {sobject_api_name}" and calls sf_api.query_all, returns list[dict]
    - find_existing_keys(client, sobject_api_name: str, key_field: str, key_values: list[str]) queries target for records matching key_values, returns set of existing key values (for skip logic)
    - remap_record_types(records: list[dict], source_rt_map: dict, target_rt_map: dict, rt_field: str = "RecordTypeId") replaces source RecordTypeId with target RecordTypeId by DeveloperName lookup; raises ValueError if a source DeveloperName has no match in target
    - two_pass_insert(client, sobject: str, records: list[dict], self_ref_field: str, name_field: str) does: Pass 1 — insert records with self_ref_field set to None, collecting {name_field_value: new_id} mapping; Pass 2 — for records that had a non-null self_ref_field, update the inserted record's self_ref_field to the new parent ID looked up by name_field
  </behavior>
  <action>
    Create tests/test_etl.py with tests for all four functions. Mock migrate.sf_api calls — do not call real Salesforce. Test that:
    - extract_records builds correct SOQL and delegates to query_all
    - find_existing_keys returns set of keys found in target (skip candidates)
    - remap_record_types swaps IDs correctly and raises ValueError on missing DeveloperName
    - two_pass_insert: pass 1 inserts with self_ref_field=None, pass 2 updates children with resolved parent IDs

    Then create migrate/etl.py implementing:
    - extract_records: Build "SELECT {','.join(fields)} FROM {sobject_api_name}", call sf_api.query_all
    - find_existing_keys: Query "SELECT {key_field} FROM {sobject_api_name} WHERE {key_field} IN ({quoted_values})", return set of values. Handle empty key_values by returning empty set.
    - remap_record_types: For each record, look up source DeveloperName via reverse of source_rt_map, then find target Id via target_rt_map[dev_name]. Mutate in place.
    - two_pass_insert: Pass 1 — copy records, null out self_ref_field, call sf_api.insert_records, build {name: new_id} map from results. Pass 2 — for records where original self_ref_field was not null, call client.{sobject}.update(new_id, {self_ref_field: parent_new_id}) where parent_new_id is looked up by the original parent's name_field value.

    For two_pass_insert, use simple-salesforce's single-record update (getattr(client, sobject).update(record_id, data)) for pass 2, since only a subset of records need the parent link set. This is simpler than a second bulk call for what is typically a small number of updates.
  </action>

  </verify>
