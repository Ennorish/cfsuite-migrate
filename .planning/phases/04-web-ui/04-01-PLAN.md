**Backend (migrate/web.py):**
1. Import `StreamingResponse` from `starlette.responses` and `asyncio`/`queue` as needed
2. Change `/api/migrate` to return a `StreamingResponse` with `media_type="text/event-stream"`:
   - Create a queue or list to collect progress events
   - Pass an `on_progress` callback to `run_migration(source_client, target_client, objects, on_progress=callback)`
   - The callback receives `(name, event, data)` — emit each as an SSE `data:` line with JSON: `{"name": "Request Flow", "event": "start", "detail": {"total": 5}}` or `{"name": "Request Flow", "event": "done", "detail": {"extracted": 5, "inserted": 3, "skipped": 2}}`
   - After migration completes, emit a final SSE event with `{"event": "complete", "status": "success", "results": results}`
   - Use a synchronous generator that yields SSE-formatted strings (FastAPI supports sync generators for StreamingResponse)
3. Keep all existing validation logic (production guard, missing params, same org check) — return regular JSONResponse for errors BEFORE streaming starts

**Frontend (migrate/static/index.html):**
1. In the migration run handler, replace the `fetch` POST with an `EventSource` or `fetch` with `ReadableStream` reader
2. Add a loading/status area inside Step 3 that shows live progress messages:
   - Show a spinner/loading indicator when migration starts
   - For each SSE event, append a status line like: `◆ Request Flows — fetching...` then update to `✓ Request Flows — 5 extracted, 3 inserted, 2 skipped`
   - Use the same dark theme styling, with status symbols: ◆ (in progress), ✓ (done)
3. On the final "complete" event, hide the loading area and show the full results table (existing behavior)

**Important:** The `on_progress` callback signature from pipeline.py is `(name, event, data)` where:
- `event="start"` with `data={"total": N}` — object migration starting
- `event="done"` with `data=result_dict` — object migration finished
